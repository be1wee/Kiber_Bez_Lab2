# Secure Tunnel

Двухстороннее шифрование с использованием двух пар ключей - 7 баллов


Написано на яп `python`, с использованием библиотеки `cryptography`.

## Описание
Есть сервер (server.py) и клиент (client.py). У каждого есть пара ключей (публичный и приватный). Устанавливается tcp соединение. 
Далее сервер и клиент обмениваются публичными ключами.  
Клиент пишет серверу сообщение, сообщение перед отправкой шифруется публичным ключом сервера. Сервер получает сообщение и расшифровывает его с помощью своего приватного ключа. Сервер пишет ответ. Шифрует его с помощью публичного ключа клиента и отправляет сообщение. Клиент получает его, расшифровывает своим приватным ключом и читает его.

## Запуск

1. Установите `cryptography`: `pip install cryptography`
2. Запустите сервер: `python server.py`
3. Запустите клиент: `python client.py`
4. Вводите сообщения, для выхода — `exit`


# Custom mTLS

Двухстороннее шифрование с использованием двух пар ключей - 10 баллов

Написано на яп `python`, с использованием библиотеки `cryptography`.

## Описание

Есть сервер (`server.py`) и клиент (`client.py`). У каждого есть свой сертификат X.509 (самоподписанный), содержащий публичный ключ, информацию о владельце и срок действия. Устанавливается TCP-соединение.  

Сервер и клиент обмениваются своими сертификатами. Каждая сторона проверяет срок действия сертификата оппонента. После успешной проверки из сертификатов извлекаются публичные ключи.  

Клиент шифрует сообщение публичным ключом сервера (из его сертификата) и отправляет. Сервер расшифровывает своим приватным ключом. Сервер шифрует ответ публичным ключом клиента и отправляет. Клиент расшифровывает своим приватным ключом.  

Таким образом, обеспечивается взаимная аутентификация (через обмен и проверку сертификатов) и конфиденциальность (асимметричное шифрование всех сообщений).

## Запуск

1. Установите `cryptography`: `pip install cryptography`
2. Сгенерируйте сертификаты: `python generate.py`
3. Запустите сервер: `python server.py`
4. Запустите клиент: `python client.py`
5. Вводите сообщения, для выхода — `exit`
